# OCR 发票工具 - 交互设计文档

## 设计原则

1. **渐进式披露**: 一步只做一件事，不展示无关信息
2. **居中卡片**: 弹出式卡片覆盖层，不离开首页语境
3. **即时反馈**: 每个操作都有明确的状态变化
4. **可回退**: 随时可返回上一步或重新开始

## 数据存储方案

### 需求分析

大盘中的「月度趋势」比对功能需要持久化存储解析结果，支持：
- 同用户的跨会话数据查询
- 按月份统计分析
- 去重（同票号不再重复计入）

### 数据模型

```typescript
// 用户表 (已有登录系统后)
interface User {
  id: string;
  email: string;
  createdAt: Date;
}

// OCR 任务表
interface OcrTask {
  id: string;
  userId: string;           // 关联用户
  createdAt: Date;
  totalFiles: number;
  successCount: number;
  failCount: number;
}

// 发票记录表（核心）
interface InvoiceRecord {
  id: string;
  userId: string;           // 归属用户（用于权限隔离）
  taskId: string;           // 关联任务

  // 唯一键：票号 + 用户ID（同一用户内去重）
  invoiceNo: string;        // 发票号码
  invoiceCode: string;      // 发票代码

  // 业务数据
  sellerName: string;       // 销售方
  buyerName: string;        // 购买方
  amount: number;           // 金额
  taxAmount: number;        // 税额
  issueDate: string;        // 开票日期（用于月度统计）
  category: string;         // 分类

  // 文件信息
  fileName: string;
  fileUrl: string;          // 存储路径（OSS/本地）

  // 状态
  status: 'parsed' | 'failed';
  failReason?: string;

  createdAt: Date;
  updatedAt: Date;

  // 软删除（用户可删除自己的记录）
  isDeleted: boolean;
  deletedAt?: Date;
}
```

### 唯一键策略

```
唯一约束: (userId, invoiceNo, invoiceCode)

为什么不是全局唯一？
- 不同用户可能上传相同发票（如公司报销多人有同一张）
- 同一用户内去重即可

处理冲突：
- 新上传：检测到重复，提示「该发票已存在于您的记录中」
- 保留最新：更新金额、日期等信息（防止早期识别有误）
```

### 月度统计查询

```sql
-- 某用户月度金额统计
SELECT
  DATE_FORMAT(issueDate, '%Y-%m') as month,
  COUNT(*) as count,
  SUM(amount) as totalAmount
FROM invoice_records
WHERE userId = ?
  AND status = 'parsed'
  AND isDeleted = false
GROUP BY DATE_FORMAT(issueDate, '%Y-%m')
ORDER BY month DESC;
```

### 权限控制

| 场景 | 处理 |
|------|------|
| 未登录用户 | OCR 识别可用，但结果不保存，刷新即清空 |
| 已登录用户 | 自动保存到个人记录，支持历史查询和比对 |
| 数据隔离 | 用户只能看到自己的发票记录 |
| 数据导出 | 支持导出个人全部记录为 Excel |

### 存储流程

```
用户上传 → OCR 识别 → 登录状态检查
                          ↓
                    ┌─────┴─────┐
                   已登录        未登录
                     ↓            ↓
              保存到数据库      仅内存存储
              生成 taskId      页面刷新丢失
                     ↓            ↓
              大盘可查历史      仅本次结果

卡片出现: 200ms ease-out, scale(0.95) → scale(1), opacity 0 → 1
步骤切换: 150ms ease-in-out, 水平滑动 + 淡入淡出
进度条: 300ms ease-out 宽度变化
数字增长: 800ms ease-out 计数动画
文件列表展开: 200ms ease height 变化
```

## 响应式

```
桌面 (>768px): 卡片 680px 居中，遮罩背景
平板 (480-768px): 卡片 90% 宽度，保持圆角
手机 (<480px): 卡片全屏，底部固定操作按钮
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 文件格式不支持 | 拖拽时显示红色边框，释放后提示「仅支持 ZIP/PDF/PNG/JPG」 |
| 文件过大 | 单个文件超过 10MB 时提示，建议压缩或分批 |
| 识别失败 | 步骤3显示「⚠ N 张识别失败」，可展开查看详情/重试 |
| 网络中断 | 步骤2显示「连接中断」，提供「重试」按钮 |
| 服务器错误 | 步骤2或3显示友好错误提示，引导重试或反馈 |

## 代码结构建议

```tsx
// 步骤枚举
type Step = 'upload' | 'processing' | 'summary' | 'details';

// 主组件
export function OcrInvoiceModal({ onClose }: { onClose: () => void }) {
  const [step, setStep] = useState<Step>('upload');
  const [files, setFiles] = useState<File[]>([]);
  const [progress, setProgress] = useState(0);
  const [result, setResult] = useState<Result | null>(null);

  return (
    <Modal onClose={onClose}>
      <StepIndicator current={step} />
      <StepContent step={step} {...{ files, progress, result }} />
      <StepActions step={step} onAction={handleAction} />
    </Modal>
  );
}
```

## 与其他工具的统一性

此交互模式可作为工具页的**标准模板**：

1. **居中卡片弹窗** - 所有工具统一使用 Modal 形式
2. **步骤指示器** - 多步骤工具必须显示进度
3. **渐进式披露** - 复杂操作分步完成
4. **即时反馈** - 每个操作都有状态变化
5. **可回退** - 用户随时可以撤销或重新开始

下一工具（如计算器）遵循：
- 单步骤工具：简化步骤指示器，只显示标题
- 结果即时显示：不需要进度条
- 保持相同卡片尺寸和动画
